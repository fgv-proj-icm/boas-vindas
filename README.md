## 1. O que é o GitHub?

O **GitHub** é uma plataforma online para armazenar, organizar e compartilhar arquivos de projetos — especialmente códigos, dados e documentações. Pense nele como um **Google Drive científico com controle de versões**: você sabe quem alterou o quê e quando.

No Lab LEPP, usamos o GitHub para:
- Centralizar códigos de análise de dados
- Compartilhar documentação dos projetos
- Colaborar em tempo real entre pesquisadores
- Manter histórico de todas as alterações

---

## 2. Conceitos essenciais

| Termo | O que significa |
|---|---|
| **Repositório (repo)** | Uma pasta de projeto com todos os seus arquivos e histórico |
| **Organização** | Grupo de repositórios compartilhados por uma equipe (ex: `fgv-proj-icm`) |
| **Branch** | Uma versão paralela do projeto para fazer testes sem afetar o principal |
| **Commit** | Um "salvar" com descrição — registra o que foi alterado e por quê |
| **Pull Request (PR)** | Proposta de incorporar alterações ao projeto principal, com revisão |
| **README** | Arquivo de apresentação do repositório (o que você está lendo agora!) |
| **Fork** | Cópia de um repositório para a sua conta, para editar livremente |
| **Clone** | Baixar uma cópia do repositório para seu computador |

## 3. Como criar sua conta

1. Acesse [github.com](https://github.com)
2. Clique em **"Sign up"** (canto superior direito)
3. Informe seu **e-mail** (recomendamos o institucional FGV)
4. Crie uma **senha** forte
5. Escolha um **nome de usuário** (ex: `joaosilva-lepp`)
6. Confirme seu e-mail através do link enviado
7. Pronto! Sua conta está criada.

> ⚠️ **Dica**: Use um nome de usuário profissional — ele aparecerá em todos os seus commits e contribuições públicas.

## 4. Como entrar na organização

Apenas membros convidados têm acesso aos repositórios privados do Lab LEPP.

**Para ser adicionado:**
1. Crie sua conta no GitHub (passo 3 acima)
2. Informe seu **nome de usuário** ao coordenador do Lab LEPP
3. Aguarde o **e-mail de convite** do GitHub
4. Clique em **"Accept invitation"** no e-mail
5. Você now é membro da organização `fgv-proj-icm`!

## 5. Como navegar pelos repositórios

1. Acesse [github.com/fgv-proj-icm](https://github.com/fgv-proj-icm)
2. Clique na aba **"Repositories"** para ver todos os projetos
3. Clique no nome de um repositório para abri-lo
4. Navegue pelas pastas clicando nos nomes
5. Clique em qualquer arquivo para visualizar seu conteúdo
6. O arquivo **README.md** de cada repositório explica o que aquele projeto faz

## 6. Como baixar arquivos

### Opção A — Baixar arquivo individual
1. Abra o arquivo desejado no repositório
2. Clique no botão **"Download raw file"** (ícone de download)

### Opção B — Baixar repositório inteiro
1. Na página principal do repositório, clique no botão verde **"Code"**
2. Selecione **"Download ZIP"**
3. Extraia o arquivo no seu computador

### Opção C — Clonar (para quem usa terminal ou R/Python)
```bash
git clone https://github.com/fgv-proj-icm/NOME-DO-REPOSITORIO.git
```

## 7. Como contribuir com arquivos

### Subir um arquivo diretamente (interface web)
1. Abra o repositório desejado
2. Navegue até a pasta correta
3. Clique em **"Add file" > "Upload files"**
4. Arraste ou selecione o arquivo
5. Escreva uma descrição do que você está adicionando (campo "Commit message")
6. Clique em **"Commit changes"**

> 💡 **Boa prática**: escreva mensagens de commit claras, ex: `"adiciona script de análise descritiva - base PNAD 2023"`

### Editar um arquivo existente
1. Abra o arquivo no repositório
2. Clique no ícone de **lápis** (✏️) no canto superior direito
3. Faça suas alterações
4. Clique em **"Commit changes"** e descreva o que foi alterado

## 8. Dúvidas frequentes (FAQ)

**❓ Não recebi o convite para a organização, o que faço?**
Verifique sua caixa de spam. O e-mail vem de `noreply@github.com`. Se não encontrar, peça ao coordenador para reenviar.

**❓ Posso usar o GitHub sem saber programar?**
Sim! Para visualizar e baixar arquivos, editar documentações e participar de discussões, não é necessário saber programar.

**❓ A diferença entre repositório público e privado?**
- **Público**: qualquer pessoa na internet pode ver
- **Privado**: somente membros convidados da organização têm acesso

Os projetos de pesquisa do Lab LEPP ficam em repositórios **privados** até a publicação.

**❓ Deletei um arquivo por engano, tem como recuperar?**
Sim! O GitHub mantém o histórico de todas as alterações. Clique em **"History"** no arquivo ou repositório e encontre a versão anterior.

**❓ Qual a diferença entre GitHub e Git?**
- **Git**: programa de controle de versão instalado no computador
- **GitHub**: plataforma online que usa o Git como base. Você pode usar o GitHub completamente pelo navegador, sem instalar nada.

**❓ Posso usar o GitHub no celular?**
Sim! O GitHub tem app para Android e iOS. É prático para visualizar arquivos e acompanhar atualizações, mas para editar códigos é mais recomendado usar o computador.

**❓ Onde peço ajuda se travar?**
Abra uma **Issue** neste próprio repositório clicando na aba "Issues" > "New issue". Descreva sua dúvida e um membro da equipe irá ajudar.
