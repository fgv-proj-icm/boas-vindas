# Tutorial 01 — Primeiros passos no GitHub

> **Para quem é este tutorial:** Membros que nunca usaram o GitHub ou que estão acessando a organização do Lab pela primeira vez.

---

## O que é o GitHub?

O **GitHub** é uma plataforma online para armazenar, organizar e compartilhar arquivos de projetos. Pense nele como um **Google Drive científico com controle de versões**: você sabe quem alterou o quê e quando.

No Lab LEPP, usamos o GitHub para:
- Centralizar códigos de análise de dados (R, Python)
- Compartilhar documentação dos projetos
- Colaborar em tempo real entre pesquisadores
- Manter histórico completo de todas as alterações

---

## Conceitos essenciais

| Termo | O que significa |
|---|---|
| **Repositório (repo)** | Uma pasta de projeto com todos os seus arquivos e histórico |
| **Organização** | Grupo de repositórios compartilhados por uma equipe |
| **Branch** | Uma versão paralela do projeto para testes sem afetar o principal |
| **Commit** | Um "salvar" com descrição — registra o que foi alterado e por quê |
| **Pull Request (PR)** | Proposta de incorporar alterações ao projeto principal, com revisão |
| **README** | Arquivo de apresentação do repositório (como este!) |
| **Clone** | Baixar uma cópia completa do repositório para seu computador |

---

## Passo 1 — Criar sua conta no GitHub

1. Acesse [github.com](https://github.com)
2. Clique em **"Sign up"** (canto superior direito)
3. Informe seu **e-mail** — recomendamos o institucional FGV ou ENAP
4. Crie uma **senha** forte
5. Escolha um **nome de usuário** (ex: `joaosilva-lepp`, `mariabonani`)
6. Confirme seu e-mail através do link enviado
7. Pronto! Sua conta está criada.

> **Dica:** Use um nome de usuário profissional — ele aparece em todos os seus commits e contribuições públicas.

---

## Passo 2 — Entrar na organização do Lab LEPP

Apenas membros convidados têm acesso aos repositórios privados do Lab.

**Para ser adicionado:**
1. Crie sua conta (passo anterior)
2. Informe seu **nome de usuário GitHub** ao coordenador do Lab LEPP
3. Aguarde o **e-mail de convite** do GitHub (vem de `noreply@github.com`)
4. Clique em **"Accept invitation"** no e-mail
5. Você agora é membro da organização!

> Se não recebeu o convite, verifique a pasta de spam. Se ainda não encontrar, peça ao coordenador para reenviar.

---

## Passo 3 — Navegar pelos repositórios

### Acessar o hub do Lab

1. Acesse a página principal deste repositório no GitHub
2. O **README.md** (esta página) mostra todos os projetos organizados por tipo
3. Clique no nome de um projeto na tabela para abrir a pasta correspondente

### Dentro de um repositório

1. Navegue pelas pastas clicando nos nomes
2. Clique em qualquer arquivo para visualizar seu conteúdo
3. O arquivo **README.md** de cada pasta explica o que aquele projeto ou seção faz
4. Clique em **"History"** (histórico) para ver todas as versões anteriores de um arquivo

## Perguntas frequentes

**Posso usar o GitHub sem saber programar?**
Sim. Para visualizar arquivos, baixar documentos, editar textos e participar de discussões, não é necessário saber programar.

**Qual a diferença entre repositório público e privado?**
- **Público**: qualquer pessoa na internet pode ver
- **Privado**: somente membros convidados têm acesso

Os projetos de pesquisa do Lab ficam em repositórios **privados** até a publicação.

**Deletei um arquivo por engano, tem como recuperar?**
Sim. O GitHub mantém o histórico de todas as alterações. Clique em **"History"** e encontre a versão anterior.

**Posso usar o GitHub no celular?**
Sim, há app para Android e iOS. É útil para visualizar e acompanhar atualizações, mas editar código é melhor no computador.
